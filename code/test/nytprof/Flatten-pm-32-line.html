295=head1 NAME
296
297Hash::Flatten - flatten/unflatten complex data hashes
298
299=head1 SYNOPSIS
300
301 # Exported functions
302 use Hash::Flatten qw(:all);
303 $flat_hash = flatten($nested_hash);
304 $nested_hash = unflatten($flat_hash);
305
306 # OO interface
307 my $o = new Hash::Flatten({
308 HashDelimiter => '->',
309 ArrayDelimiter => '=>',
310 OnRefScalar => 'warn',
311 });
312 $flat_hash = $o->flatten($nested_hash);
313 $nested_hash = $o->unflatten($flat_hash);
314
315=head1 DESCRIPTION
316
317Converts back and forth between a nested hash structure and a flat hash of delimited key-value pairs.
318Useful for protocols that only support key-value pairs (such as CGI and DBMs).
319
320=head2 Functional interface
321
322=over 4
323
324=item $flat_hash = flatten($nested_hash, \%options)
325
326Reduces a nested data-structure to key-value form. The top-level container must be hashref. For example:
327
328 $nested = {
329 'x' => 1,
330 'y' => {
331 'a' => 2,
332 'b' => 3
333 },
334 'z' => [
335 'a', 'b', 'c'
336 ]
337 }
338
339 $flat = flatten($nested);
340 use Data::Dumper;
341 print Dumper($flat);
342
343 $VAR1 = {
344 'y.a' => 2,
345 'x' => 1,
346 'y.b' => 3,
347 'z:0' => 'a',
348 'z:1' => 'b',
349 'z:2' => 'c'
350 };
351
352The C<\%options> hashref can be used to override the default behaviour (see L</OPTIONS>).
353
354=item $nested_hash = unflatten($flat_hash, \%options)
355
356The unflatten() routine takes the flattened hash and returns the original nested hash (see L</CAVEATS> though).
357
358=back
359
360=head2 OO interface
361
362=over 4
363
364=item $o = new Hash::Flatten(\%options)
365
366Options can be squirreled away in an object (see L</OPTIONS>)
367
368=item $flat = $o->flatten($nested)
369
370Flatten the structure using the options stored in the object.
371
372=item $nested = $o->unflatten($flat)
373
374Unflatten the structure using the options stored in the object.
375
376=back
377
378=head1 OPTIONS
379
380=over 4
381
382=item HashDelimiter and ArrayDelimiter
383
384By default, hash dereferences are denoted by a dot, and array dereferences are denoted by a colon. However
385you may change these characters to any string you want, because you don't want there to be any confusion as to
386which part of a string is the 'key' and which is the 'delimiter'. You may use multicharacter strings
387if you prefer.
388
389=item OnRefScalar and OnRefRef and OnRefGlob
390
391Behaviour if a reference of this type is encountered during flattening.
392Possible values are 'die', 'warn' (default behaviour but warns) or a coderef
393which is passed the reference and should return the flattened value.
394
395By default references to references, and references to scalars, are followed silently.
396
397=item EscapeSequence
398
399This is the character or sequence of characters that will be used to escape the hash and array delimiters.
400The default escape sequence is '\\'. The escaping strategy is to place the escape sequence in front of
401delimiter sequences; the escape sequence itself is escaped by replacing it with two instances.
402
403=item DisableEscapes
404
405Stop the escaping from happening. No escape sequences will be added to flattened output, nor interpreted on the way back.
406
407B<WARNING:> If your structure has keys that contain the delimiter characters, it will not be possible to unflatten the
408structure correctly.
409
410=back
411
412=head1 CAVEATS
413
414Any blessings will be discarded during flattening, so that if you flatten an object you must re-bless() it on unflattening.
415
416Note that there is no delimiter for scalar references, or references to references.
417If your structure to be flattened contains scalar, or reference, references these will be followed by default, i.e.
418C<'foo' =E<gt> \\\\\\$foo>
419will be collapsed to
420C<'foo' =E<gt> $foo>.
421You can override this behaviour using the OnRefScalar and OnRefRef constructor option.
422
423Recursive structures are detected and cause a fatal error.
424
